# Jobs - Plain English Overview

Jobs are the main containers for client work in Net Net. A Job can be a Project or a Retainer, and each type has a different performance story.

## Project Performance

Projects are finite. They have a defined start, a target finish, an approved scope, and a timeline.

Project Performance is one report against the approved plan. It answers:

- What did we plan?
- What changed?
- How many hours were used?
- How much work remains?
- Where did delivery drift?

Projects are timeline-based, so the performance view should feel like one complete job report.

## Retainer Performance

Retainers are cycle-based. They have a start date and may have an end date.

Active or open-ended Retainers show `-` for end date. Fixed-term or completed Retainers show their actual or derived end date.

Retainer reporting begins with the Retainer start month. Month navigation cannot move before that start month. If an end date exists, navigation cannot move after the end month. If there is no end date, navigation can move through the latest available or current cycle.

Jobs Report Retainer detail defaults to the Retainer start month because it is a historical lookup. The individual Job Performance tab defaults to the current or latest active cycle for active Retainers, and to the final available cycle for completed or archived Retainers.

The Retainer Summary stays visible above the selected month. The summary explains the overall pattern across the Retainer. Each selected month has its own monthly report for plan, actual hours, remaining capacity, variance, change orders, deliverables, and service type mix.

Retainer reports support both fixed monthly work and one-off work. A one-off task appears only in the month or months where it belongs. For example, monthly Plugin Updates can appear every cycle, Emergency Diagnostics can appear in one month, and Landing Page Fixes can span two months until completed.

## Date Filters

Jobs Report date filters should follow the shared Net Net calendar picker pattern. Plain date fields are acceptable in the prototype until calendar picker cleanup is handled as a focused task.
